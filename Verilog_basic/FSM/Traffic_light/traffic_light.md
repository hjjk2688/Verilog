# 보안 시스템 FSM (유한 상태 머신) 구조

<img width="769" height="386" alt="image" src="https://github.com/user-attachments/assets/d0fe39d7-84b8-4d8b-ada8-a754feda38d8" />


   - 초기 상태: 잠금 (ARMED)

   - 주요 변수:

       - 비밀번호: 1100

       - 센서 감지: sensors 값이 0이 아님

       - 시간 초과: count_done이 1이 됨 (30초 경과)

​

  1. 잠금 (ARMED) 상태

   - 설명: 시스템이 무장되어 외부 침입(센서)을 감시하는 기본 상태입니다.

   - 전환 조건:

       - 만약 [센서가 감지되면] → 대기 (WAIT_DELAY) 상태로 전환합니다.

       - 만약 [비밀번호(`1100`)를 입력하면] → 열림 (DISARMED) 상태로 전환합니다.

       - 그 외의 경우 → 잠금 (ARMED) 상태를 유지합니다.

  2. 열림 (DISARMED) 상태

   - 설명: 시스템이 해제되어 경보가 울리지 않는 안전한 상태입니다.

   - 전환 조건:

       - 만약 [비밀번호(`1100`)를 입력하면] → 잠금 (ARMED) 상태로 전환합니다.

       - 그 외의 경우 → 열림 (DISARMED) 상태를 유지합니다.

  3. 대기 (WAIT_DELAY) 상태

   - 설명: 센서 감지 후, 사용자에게 비밀번호를 입력할 30초의 시간을 주는 상태입니다.

   - 전환 조건:

       - 만약 [30초 안에 비밀번호(`1100`)를 입력하면] → 열림 (DISARMED) 상태로 전환합니다.

       - 만약 [30초가 지나면 (시간 초과)] → 알람 (ALARM) 상태로 전환합니다.

       - 그 외의 경우 → 대기 (WAIT_DELAY) 상태를 유지합니다.

  4. 알람 (ALARM) 상태

   - 설명: 침입이 확정되어 경보(alarm_siren)를 계속 울리는 상태입니다.

   - 주요 동작: alarm_siren 출력을 1로 만들어 경보를 울립니다.

   - 전환 조건:

       - 만약 [비밀번호(`1100`)를 입력하면] → 열림 (DISARMED) 상태로 전환하고 경보를 끕니다.

       - 그 외의 경우 → 알람 (ALARM) 상태를 유지하며 경보를 계속 울립니다.
